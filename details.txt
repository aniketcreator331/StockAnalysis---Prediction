STOCK DASHBOARD PROJECT - COMPONENT DETAILS AND WORKFLOW EXPLANATION

This document provides a comprehensive overview of the file structure, technology stack, logic, and development workflow for the full-stack Stock Analysis Dashboard.

==================================================
1. TECHNOLOGY STACK OVERVIEW
==================================================

Backend:
- Python 3.x
- FastAPI (Web framework for high-performance REST APIs)
- Uvicorn (ASGI server)
- yfinance / Alpha Vantage (APIs to fetch stock market live/historical data)
- TensorFlow / Keras (LSTM Neural Network for AI future price forecasting)
- Pandas, Numpy, Scikit-learn (Data processing and AI scaling)

Frontend:
- JavaScript (JSX)
- React (Component-based UI library)
- Vite (Fast development build tool)
- Tailwind CSS (Utility-first styling approach for modern UI)
- React Router DOM (Navigation between pages)
- Lucide React (Modern SVG Icon set)
- ApexCharts & Recharts (Data visualization libraries for candlesticks and line/bar graphs)

==================================================
2. BACKEND FILE DETAILS (Directory: /backend)
==================================================

- main.py
  The entry point for the FastAPI server. It defines API endpoints including live dashboard info (`/api/dashboard`), historical charts (`/api/chart`), AI predictions (`/api/predict`), top tech stock recommendations (`/api/recommendations`), and triggering model retraining (`/api/train_model`).

- data_fetcher.py
  Contains the core logic for communicating with external APIs (like Yahoo Finance). Extracts pricing (Open, High, Low, Close, Volume), calculates daily growth, and formats historical timeseries arrays.

- prediction_model.py
  Contains the AI implementation. Uses an LSTM (Long Short-Term Memory) sequential AI model to learn from previous stock prices and output predictions for future intervals (e.g., the next 30 minutes).

- stock_api.py
  A supplementary module to calculate unique insights like volatility and structure recommendations array.

==================================================
3. FRONTEND LOGIC & COMPONENTS (Directory: /frontend/src)
==================================================

A. Root Files:
- main.jsx & App.jsx: The React entry points. `App.jsx` handles global state, header logic (currency selector, dark mode, help modal), global context wrappers (`CurrencyProvider`, `UserDataProvider`), and the `Router` setup for navigation.

B. Pages (`/pages`):
- Dashboard.jsx: The main core dashboard. Features a ticker search bar, metric cards summarizing the stock, a candlestick graph of the stock's history, and a price alerts card.
- PredictionPage.jsx: A dedicated page for AI. Displays an interactive curve mapping the historical data seamlessly into forecasted target prices generated by the LSTM algorithm.
- ComparisonPage.jsx: Analyzes a pre-arranged basket of top tech equities, calculating daily growths side-by-side using an animated Bar Chart.
- CompareTwoPage.jsx: Enables a user to input any two varying stock tickers, lining up their financial parameters (prices, growth, volume, volatility) head-to-head.
- TickersPage.jsx: A useful directory referencing the common ticker abbreviations to assist users unfamiliar with market keywords.

C. Visual Dashboard Components (`/components` & `/charts`):
- Sidebar.jsx: Primary navigation menu on the left side of the screen. Later updated to contain user tracking info (Followed Stocks, View History, and Search History).
- MetricCard.jsx: Modular UI widgets used across the app to display a data point alongside a colored graphical icon and growth percentages.
- PriceAlertsCard.jsx: A widget containing form inputs to set a condition (Above/Below) and a target price limit. Contains logic to push a triggered notification if that boundary is crossed. 
- HelpModal.jsx: An animated window accessed from the header containing instructional steps for the website.
- CandlestickChart.jsx (ApexCharts): Renders professional multi-axis price arrays for the dashboard.
- PredictionLineChart.jsx / ComparisonChart.jsx (Recharts): Custom SVGs displaying continuous forecasting paths and comparative bars respectfully.

D. Global State Management (`/contexts`):
- CurrencyContext.jsx: Queries an ExchangeRate API in the background. Exposes `formatPrice()` functions, allowing every number on the screen to seamlessly convert back and forth between USD, INR, EUR, GBP, and JPY based on header selection.
- UserDataContext.jsx: Manages the localStorage tracking variables (`followedStocks`, `searchHistory`, `viewHistory`, `priceAlerts`). Binds these states globally so the Sidebar updates dynamically if a user searches on the Dashboard.

E. Services (`/services`):
- api.js: Abstracts JS fetch protocol and base URLs, providing neat asynchronous bridging functions linking React frontend actions directly to Python FastAPI routes.

==================================================
4. STEP-BY-STEP DEVELOPMENT WORKFLOW
==================================================
Here is the chronological order of how this project evolved:

Step 1: Environmental Setup & Baseline Configuration.
Built the basic Python backend via FastAPI and formatted a standard Vite+React JS project. Handled preliminary CORS errors.

Step 2: Core Data Fetching & Dashboard Structure.
Wrote Python endpoints to gather data via yfinance. Created `Dashboard.jsx`, the Sidebar, basic `MetricCards`, and routed data into `CandlestickChart.jsx`.

Step 3: Machine Learning & Predictions.
Designed `prediction_model.py` and injected standard LSTM training logic. Wired this data successfully into `PredictionPage.jsx` plotting actual vs forecasted graphs using `Recharts`.

Step 4: Market Broadening & Feature Comparison.
Added endpoints for mass recommendations. Constructed `ComparisonPage.jsx` and the highly modular `CompareTwoPage.jsx` letting users fetch distinct timelines of disparate stocks simultaneously.

Step 5: Aesthetics & Layout Solidification.
Enhanced the entire application UI utilizing intricate Tailwind utility classes — dark mode toggling, backdrop-blur borders, and glassmorphic modal effects yielding a modern, premium UX. Added `TickersPage.jsx` and upgraded the Footer design.

Step 6: Currency Multi-Support Integration.
Implemented `CurrencyContext.jsx` handling dynamic cross-component rate mapping so users can accurately view Live Markets in localized values like Rupees (INR).

Step 7: User Personalization & Tracking Context.
Introduced `UserDataContext.jsx` to map user history cleanly into their `localStorage` state. Connected "Follow Stock" pins, "Viewed Lists" and "Recent Searches" seamlessly into Sidebar integration.

Step 8: Help System & Complex Condition Alerts:
Created an explicit `HelpModal.jsx` accessible on a global scale. Finally, injected a customizable `PriceAlertsCard` directly adjacent to the main graphs allowing intelligent boundary tracking rules.
